Topic Markdown Format
2026-06-27 v1 Author's guide
01Shape of the file

One .md file becomes one topic with any number of subtopics. Upload it from home → Add topic.

---
title: My Topic
icon: "📦"
description: One-line summary shown on the card.
---

# First Subtopic

Theory markdown goes here.

## Checklist
- [ ] First thing to learn
- [ ] Second thing to learn

# Second Subtopic

More theory…

## Checklist
- [ ] Another item
02Frontmatter (topic info)

A --- fenced block at the very top. Flat key: value pairs only.

FieldRequiredNotes
titleyesThe topic name shown on the card and headings.
iconnoA single emoji, e.g. "📦". Quotes optional.
descriptionnoOne-line summary shown on the topic card.
03Subtopics
  • Every # (H1) heading starts a new subtopic; the heading text is its title.
  • Subtopics appear in the order they're written.
  • A topic needs at least one subtopic.
04Checklists

Inside a subtopic, add a ## Checklist heading followed by a task list. Each - [ ] line is one tickable item.

## Checklist
- [ ] Understand the idea
- [ ] Do one practice problem
The ## Checklist section is removed from the rendered theory so it isn't shown twice — the interactive checklist renders it instead. A subtopic may omit the checklist (it just renders theory).
05Supported markdown in the theory

The body of each subtopic supports the same features as built-in lessons:

  • Headings (##–######), bold, italic, lists, tables, links, images.
  • Code blocks with syntax highlighting (fenced with a language, e.g. js, python).
  • Callouts — > [!NOTE], plus [!TIP], [!WARNING], [!CAUTION], [!IMPORTANT].
  • Diagrams — a ```mermaid fenced block.
06Common errors
MessageFix
Missing title in frontmatter.Add a --- block at the top with title:.
No subtopics found…Add at least one # H1 heading.
Subtopic … has no checklist items.Warning only — add a ## Checklist if you want tracking.
07Uploading

Home → Add topic → choose your .md file or paste its contents. A preview shows the topic, its subtopics, and item counts before you confirm.

Your uploaded topics are saved in this browser and included in your export/import backup (Stats → Your data). Grab the starter at public/templates/topic-template.md or via the "Download the template" link in the upload dialog.
